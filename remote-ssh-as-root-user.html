Remote SSH as root user

By default, the root user is disabled, but we need it for use with our SFTP client later in the project An SFTP client is a program that allows us to transfer files between two computers using the same secure protocol used in SSH When we are working on the Raspberry Pi using the terminal emulator and SSH, we can simply switch to the root user with the "sudo" command:
JavaScript
Copy
sudo su
​
This means that the root (or "super user", hence "su") user is available, but is only accessible via the "sudo" command It is not enabled for remote access, such as SSH or SFTP Once you become "super user" with "sudo", you can work as if you were logged in as "root", and access parts of the file system that are only permitted to the root user, or edit files owned by root To exit the super user and go back to the normal "pi" user, type:
JavaScript
Copy
exit
​
